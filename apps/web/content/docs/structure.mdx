## Key Files

* **`apps/web/src/middleware.ts`**: Contains route protection logic and redirects
* **`apps/web/src/routes.ts`**: Defines core routes for consistent navigation
* **`packages/db/src/schema.ts`**: Database schema definitions
* **`apps/web/next.config.ts`**: Next.js configuration
* **`source.config.ts`**: Fumadocs configuration
* **`env.example`**: Environment variable template

## Code Organization Best Practices

When using AeloKit, follow these code organization best practices:

<Steps>
  <Step>
    ### Group by feature, not by type

    Keep related files together, regardless of their type.
  </Step>

  <Step>
    ### Keep components focused and small

    Each component should have a single responsibility.
  </Step>

  <Step>
    ### Use barrel exports wisely

    Create `index.ts` files to re-export components from a directory, making imports cleaner.

    ```tsx
    // components/ui/index.ts
    export * from './Button';
    export * from './Card';
    export * from './Input';

    // Usage
    import { Button, Card, Input } from '@/components/ui';
    ```
  </Step>

  <Step>
    ### Separate client and server components

    Use the `'use client'` directive at the top of client components, and keep server components as the default.
  </Step>
</Steps>
