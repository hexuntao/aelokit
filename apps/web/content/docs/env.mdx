AeloKit requires some environment variables to run properly. This document explains how to set up these environment variables.

Create an environment variable file (`.env`) in the project root directory, and **never commit it to version control**.

You can copy the `.env.example` file as a starting point:

```bash
cp env.example .env
```

## Environment Variable Architecture

AeloKit uses the `@repo/env` package for environment variable validation:

- `packages/env/src/server.ts` - Server-side variables
- `packages/env/src/client.ts` - Client-side variables (`NEXT_PUBLIC_*`)
- `packages/env/src/shared.ts` - Shared variables

<Callout type="info">
  Application code should read environment variables through `@repo/env/server` or `@repo/env/client`, instead of directly using `process.env`.
</Callout>

## Core Configuration

| Variable | Description | Required |
| --- | --- | --- |
| `NEXT_PUBLIC_BASE_URL` | The URL of the website, e.g. `http://localhost:3000` for development, `https://your-domain.com` for production | ✅ |
| `NEXT_PUBLIC_PAYMENT_PROVIDER` | Payment provider: `stripe` or `creem` | ✅ |

## Validating Environment Variables

### Runtime Validation

Environment variables are automatically validated when starting the development server:

```bash
pnpm dev
```

### Manual Validation

Run the validation command:

```bash
pnpm check:env
```

### Skipping Validation

You can skip validation in CI/build environments:

```bash
SKIP_ENV_VALIDATION=true pnpm build
```

<Callout type="warn">
  Do not use `SKIP_ENV_VALIDATION=true` in production, as this will skip all environment variable validation.
</Callout>
