## 什么是 AeloKit？

欢迎使用 AeloKit，**一个 OpenAI 风格的开源全栈套件**，用于构建可自托管、生产就绪的 AI 聊天和 Agent 工作空间。

从样板代码到生产环境——快速构建您的 AI 工作空间。

无论您是初学者还是经验丰富的开发者，本文档都将指导您完成项目的配置、开发和部署。

本文档假设您对 Next.js 和 Monorepo 有一定的了解。如果您不熟悉，请参考[官方文档](#官方文档)，因为我们不会在本文档中涵盖基础知识。

让我们开始吧！

## Monorepo 结构

AeloKit 采用 **Turborepo + pnpm workspace** 的 Monorepo 架构，将不同领域的逻辑拆分到独立的包中：

<Files>
  <Folder name="apps">
    <Folder name="web">
      <File name="完整的 SaaS 应用程序" />
    </Folder>
  </Folder>
  <Folder name="packages">
    <File name="config/ - 共享配置和类型" />
    <File name="env/ - 环境变量验证" />
    <File name="i18n/ - 国际化路由和消息" />
    <File name="db/ - Drizzle 数据库层" />
    <File name="auth/ - 认证核心" />
    <File name="payment/ - 支付领域包" />
    <File name="credits/ - 积分领域包" />
    <File name="mail/ - 事务邮件包" />
    <File name="newsletter/ - 邮件订阅包" />
    <File name="notification/ - 系统通知包" />
    <File name="storage/ - 对象存储包" />
    <File name="analytics/ - 统计分析包" />
    <File name="shared/ - 共享工具函数" />
  </Folder>
</Files>

<Callout type="info">
  Monorepo 结构的优势：清晰的领域边界、代码复用、独立版本管理、更好的类型安全和更易于维护。
</Callout>

## 快速开始

```bash
# 安装依赖
pnpm install

# 复制环境变量配置
cp env.example .env

# 启动开发服务器
pnpm dev
```

打开 Next.js 打印的本地 URL 即可访问 Web 应用。

<Callout type="warn">
  请勿提交密钥。应用程序代码应通过 `@repo/env/server` 或 `@repo/env/client` 读取业务环境变量，而不是直接使用 `process.env`。
</Callout>

## 常用命令

```bash
# Monorepo 级命令
pnpm dev          # 启动开发服务器
pnpm build        # 构建所有包
pnpm lint         # 代码检查
pnpm typecheck    # 类型检查

# Web 应用命令
pnpm --filter @repo/web dev      # 启动 web 开发服务器
pnpm --filter @repo/web build    # 构建 web 应用

# 数据库命令
pnpm --filter @repo/db db:generate   # 生成迁移文件
pnpm --filter @repo/db db:migrate    # 执行迁移
pnpm --filter @repo/db db:studio     # 打开 Drizzle Studio
```
