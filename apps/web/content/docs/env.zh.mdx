AeloKit 需要一些环境变量才能正常运行，本文档解释了如何设置这些环境变量。

在项目根目录创建一个环境变量文件（`.env`），并且**永远不要将其提交到版本控制**。

您可以复制 `.env.example` 文件作为起点：

```bash
cp env.example .env
```

## 环境变量架构

AeloKit 使用 `@repo/env` 包进行环境变量验证：

- `packages/env/src/server.ts` - 服务端变量
- `packages/env/src/client.ts` - 客户端变量（`NEXT_PUBLIC_*`）
- `packages/env/src/shared.ts` - 共享变量

<Callout type="info">
  应用代码应通过 `@repo/env/server` 或 `@repo/env/client` 读取环境变量，而不是直接使用 `process.env`。
</Callout>

## 核心配置

| 变量 | 描述 | 必需 |
| --- | --- | --- |
| `NEXT_PUBLIC_BASE_URL` | 网站的 URL，例如 `http://localhost:3000` 用于开发环境，`https://your-domain.com` 用于生产环境 | ✅ |
| `NEXT_PUBLIC_PAYMENT_PROVIDER` | 支付提供商：`stripe` 或 `creem` | ✅ |

## 验证环境变量

### 运行时验证

启动开发服务器时会自动验证环境变量：

```bash
pnpm dev
```

### 手动验证

运行验证命令：

```bash
pnpm check:env
```

### 跳过验证

在 CI/build 环境中可以跳过验证：

```bash
SKIP_ENV_VALIDATION=true pnpm build
```

<Callout type="warn">
  不要在生产环境中使用 `SKIP_ENV_VALIDATION=true`，这会跳过所有环境变量验证。
</Callout>
